# Customer Detail Screen

## What to build

Build the customer detail screen so users can review and edit the core customer
profile in one place.

The screen should show current customer identity, communication, and lifecycle
fields and provide editing behavior consistent with the API validation rules.

## Acceptance criteria

- [ ] A user can open a customer detail screen from a customer record.
- [ ] The detail screen shows display name, customer type, status, legal name, document data, email, and phone.
- [ ] The user can update editable core fields from the detail screen.
- [ ] Validation errors are shown near the relevant fields.
- [ ] The saved detail view reflects persisted API state after update.
- [ ] Tests cover customer detail loading and core field editing where the test stack supports it.
